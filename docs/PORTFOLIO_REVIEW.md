## Что стоит сделать перед публикацией на GitHub

1. Добавить скриншот главного окна в `docs/` и вставить его в README.
2. Проверить запуск на чистой Windows-машине или в новой виртуальной среде.
3. Выполнить:

   ```bash
   python -m pytest
   python -m ruff check .
   python -m mypy .
   pyinstaller timer_3.spec
   ```

4. Создать короткое описание репозитория:

   ```text
   Desktop countdown timer on Python/PyQt6 with voice notifications, JSON settings and PyInstaller build.
   ```

## Что не стоит показывать в портфолио

- рабочие JSON-файлы с абсолютными путями вида `C:/PycharmProjects/...`;
- папки `.venv`, `build`, `dist`, `__pycache__`;
- временные файлы экспериментов.
