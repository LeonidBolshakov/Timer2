# Архитектура Timer 2

## Основные компоненты

`timer_3.py` отвечает за главное окно, валидацию ввода и связывание UI с логикой.

`clock.py` содержит сценарий обратного отсчёта: уменьшение времени, обновление отображения, 
запуск голосовых сообщений, финальные звуковые сигналы и событие завершения.

`precise_timer.py` инкапсулирует `QTimer` и компенсирует накопленный дрейф, 
чтобы таймер не уходил заметно от реального времени.

`inform.py` отвечает за голосовое оповещение через `pyttsx3` и проигрывание финальной мелодии через `pygame`.

`tunes_model.py`, `tunes_dto.py`, `tunes_mapper.py`, `tunes_storage.py` образуют слой настроек:

- `TunesDTO` — JSON-совместимое представление;
- `TunesModel` — рабочая модель приложения;
- mapper — преобразование между DTO, моделью и JSON;
- storage — чтение, запись, fallback на значения по умолчанию.

## Почему настройки вынесены в профиль пользователя

Каталог программы может быть недоступен для записи после установки или сборки в `.exe`. 
Поэтому рабочие настройки сохраняются в `%APPDATA%\Timer_3`.

## Что можно улучшить дальше

- перевести проект на пакетную структуру `src/timer_3_app/`;
- скомпилировать `.ui` в Python-классы или перейти на Qt Resource System;
- добавить GUI-тесты через `pytest-qt`;
- добавить GitHub Actions для проверки `pytest`, `ruff`, `mypy`;
- заменить строковые callback-имена в `Clock` на Enum или Qt-сигналы.
